File Reader

Requirements

This requires a server instance to run.

To install a local server:

npm install -g http-server

Ensure Environment Variable Path value contains the following (replace [username]):

C:\Users\[username]\AppData\Roaming\npm C:\Users\[username]\AppData\Roaming\npm\node_modules\http-server

change the selected drive (if necessary)

[drive letter]:

change the directory to the root of your project.

cd C:\project\directory

instantiate a server with the following:

http-server -c-1

Rules

The <script> tag that calls fileReader.read( filePath, destinationElement, options ), must always appear after the "destinationElement" in your HTML.

fileReader (OBJECT)

fileReader.read( filePath, destinationElement, options )

filePath // a valid path to the file, including the file name (STRING)

destinationElement // the element where you want the contents of the file to display (ELEMENT OBJECT)

options // optional settings for the element (OBJECT)

  • { headerRow: true } // first record or row of tabular data written in <th> tags
  • { headerRow: false } or omit object // first record or row of tabular data written in <td> tags

Ordered Elements (Example 1)

If you want your elements to display in a certain order, make sure to pass them unique tags.

Following this <p> tag are some empty <div> tags. One of the easier and more accurate methods is to use IDs which is the example used here.

Unordered Elements (Example 2)

If you don't care what order your elements display in and you just want the fastest things to display first, you can load them into a single element. They may load in the order defined in your <script> tag, but this is not guaranteed.
